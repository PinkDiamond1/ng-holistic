Reasoning

The purpose of this library is allow developers to create components (data lists in particular) not by implementing them with html, styles and code but by describing (defining) them. Then it is concern of the UX librariry to generate these components, this way we effectively separate core (business) logic of the application from the UI impelmentation.

Tables and lists

With help of this library you define (describe) the data list representation with following features

  • Define columns
    • Define title of the column
    • Define data binding to the rows values
    • Define if columns sortable
    • Provide column value formatter
    • Define classes to apply to the columns
    • Provide interface to generate custom components inside the columns
  • Define filters
  • Define default sort column
  • Define row actions for rows
  • Deine aggregation row
  • Deine selectable rows mode
  • Define expandable children rows section for rows
  • Deine drag and drop mode

Benefits

The major benefits from using this library could achive large data driven applications with a lot of tables and lists. Using this library you could easily and reliable describe desired table representations without concern how these data will be rendered. Library provides strict set of API (aka description DSL) to render UI components, this allows large teams to stay sync on how to create and use data lists inside application, no need to copy / paste html css code when create new components or invent some base classes which abstracts common list implementation for the application.
