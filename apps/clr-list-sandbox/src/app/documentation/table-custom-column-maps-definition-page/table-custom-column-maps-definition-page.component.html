Custom columns definition map

According to Domain Design principles in order to make application implementation more robust hence easier to implement and understand, we should define some domain model and operate it inside our application.

    In context of this library this means following:
  • First of all we define the set of common column types specific for the application (we could call it domain model of the app).
  • For each column type we implement specific component associated with it, for example there is could be CardNumberColumnComponent which display card number and icon of the card's scheme (visa / master / etc), or some PersonColumnComponent which will display highlighted name of the user and popup which will show additional user info. These components should be developed by design team and verified by business domain experts.
  • We configure link between column type and associated component.
  • After this we could use these column types in any table definition across whole application just describing this column in table definition object. So when domain expert asks developer to add column with CardNumber type to some table there is no ambiguity how it should be done and what UI to use for this.
