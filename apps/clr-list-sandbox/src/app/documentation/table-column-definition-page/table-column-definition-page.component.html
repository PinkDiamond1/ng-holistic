Table.Column

Example
Property Type Description
id string Column identifier, when row provided to the table from json object, value from the row[id] property will be provided for this column if bind is not defined.
bind (optional) (row: Table.Row) => any This function provides value which must be displayed inside the cell.
  • Given generated row this method must return some value to be displayed as cell content.
  • If this property is not defined then default binding will be used, it will take value from row property with name id.
  • Before displaying the bound value could be formatted.
title string Column title
sort (optional) string | boolean (default false)
    Define column as sortable
  • When column sortable it will have underline under the title, to mark column as sortable
  • When user click this column, table will be reloaded with appropriate sort parameter
  • The sort field in data provider mapState argument will have name equal to the column id
  • If this value is a string, column considered sortable and sort field in data provider will have one value instead of column id
    • cls (optional) string
      Class to apply on content of the column value
    • It could be just a sting - name of the class
    • It could be function which will set class of cell content based on condition
      • conditional function has cell value as a first param and optional second param - row itself
      • result of the function must be either class to apply to the cell content or undefined to apply nothing
    • if you return object with `cls: 'class name'` from format property it will be joined to this value
    format (optional) (val: any, row: any) => string | FormatFun
      function or reference name from configured default formatters map used to format cell value to displayed string
    • the function first arg is the cell value and second optional is the row object itself
    • function could return simple string object, it will be considered value to display
    • function could also return ` val: 'val', cls: 'cls' ` object. It will be considered : `val` as a formatted value to display and `cls` is a class to apply to cell content
    • it is recommended to use default formatters map instead of defining formatter function for each table. Use HLC_CLR_TABLE_CELL_FORMAT_MAP provider to define map of default formatters which could be used across the application simple by using reference name.

    Default formatters

    You could provide default formatters for table definitions across the application. If a lot of cells in various tables use the same formatter it is a good idea to configure this formatter for whole application and then reference it in table cell format definitions by providing reference name. Configure formatters maps via HLC_CLR_TABLE_CELL_FORMAT_MAP provider.
