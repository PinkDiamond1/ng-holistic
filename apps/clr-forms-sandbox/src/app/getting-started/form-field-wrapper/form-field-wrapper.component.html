Form field wrapper

Usually when control is placed on form it supposed to have some accompanied features which doesn't belong to it directly but which create context of that control. The inextricable example of such context is labels and validation errors. So when form control is rendered it will be placed inside container component. Default 'ng-holistic' container component has following features.
  • Label
  • Required field hint
  • Errors hint
See source of the default wrapper here.
Any input / output property of the wrapper component could be initialized by the same props rules as a form field control

Custom wrappers

For you application you could define global custom field wrapper, it will be applied to the all holistic forms of the app Example.
Also you could override global field wrapper for particular form by using HLC_FORM_FIELD_WRAPPER provider on the form Example.

Required fields hint

Default implementation of the input wrapper could display hints for required fields. When control has 'required' validator on it we could hint user that this field is required or optional Example.
