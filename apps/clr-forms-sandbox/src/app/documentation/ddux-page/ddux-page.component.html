Domain Driven UX

Domain Driven Design is wildly used approach to software development. You could consider this library as an attempt to use the same principles to define user interfaces inside frontend data driven applications. Domain expert could describe customer invoice form in following terms
Label Input type Validators Description
First Name Text alpha, min-length:3, max-length:100 Customer first name
Last Name Text required, alpha, min-length:3 max-length:100 Customer last name
Order Number Number required, min-length:5, max-length:10 Order number
Order Date Date required, past-only Date when order was made
Card Number Number required, length:12 Credit card number
Amount Number required Amount in USD (2 decimal digits)
Address String Address of delivery, empty if self pick up

Using this table developer could define form almost 1:1 reflecting domain expert definition
