Form field common properties

Every form declaration contains some group with fields list. The items from this list describe the set of components and their properties which will be rendered and bound to the form value. Here base class declaration for any FromField.

FormField common properties

Property Description
id Identifier of the generated input control, id plays crucial role in form's binding, this property will be automatically created on formGroup and input's value binding will be sync; all changes on the input control will be reflected on the formGroup and vice versa, if generated control implements ControlValueAccessor interface.
kind Type of the generated control, there is number of the most common controls already included in the library, you could add any custom component to be generated on the form.
props Contains set of values for generated component's Input properties. For example 'placeholder' property exists on the component associated with TextField kind, so the component 'placeholder' property will be initialized from value of the 'props.placeholder' field. 'label' property exists on the control wrapper, so it also will be initialized there.
label Convention property, the same as props.label, since supposedly each field definition will have 'label' property and it would be inconvenient to define each time it in props object.
validators Contains array of regular angular validators to be applied to the generated form control, works the same way as in standard FormGroup control initialization. Also this value could be observable with the same array of validators as emitted value, this case control validators will be updated accordingly to the emitted value.
hidden Observable - hide / show control on the form, when controls is hidden it value will be removed from formGroup automatically. Example
value Observable - defines the current value of the control, the way to implement form fields values recalculation. Example
