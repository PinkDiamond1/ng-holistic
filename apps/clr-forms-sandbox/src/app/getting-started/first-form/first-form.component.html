Create basic form

{{ pageSampleFormCode }} Here we create form with single text input. There is three most important properties of the field which define generated input control.
Property Description
id Identifier of the generated input control, id plays crucial role in form's binding, this property will be automatically created on formGroup and input's value binding will be sync; all changes on the input control will be reflected on the formGroup and vice versa, if generated control implements ControlValueAccessor interface.
kind Type of the generated control, there is number of the most common controls already included in the library, you could add any custom component to be generated on the form.
props Contains set of values for generated component's Input properties. In the example above 'placeholder' property exists on the component associated with TextField kind, so the component 'placeholder' property will be initialized from value of the 'props.placeholder' field. 'label' property exists on the control wrapper, so it also will be initialzied from here.

Set rendered component properties via Observable

As demonstarted above every @Input() property on the rendered component could be initialzied from the property with the same name on 'props' field of the form definition object. There is another feature, it is possible to use not just scalar value for initialziation of the components properties but 'observale' stream, which case the component property will be updated every time when stream emits some value. {{ pageSampleFormObservableProp }}

Handle Output events of the rendered component

There is way to hook on the rendered component's @Ouptut() events. Output events is just an property with Observable value, so it is possible to subscribe on it and recieve updates when them happens. For this purpose create field on the 'props' object (same as for Input ones) but value of the property must be a Subject object, this way you could subscribe on this stream and recieve updates about any required event. {{ pageSampleOutputForm }} In example above component associated with 'TextField' kind has 'valueChange' output event, also 'valueChange' property defined on 'props' object definition, with value of type Subject, hence this subject will recieve updates from component event.
