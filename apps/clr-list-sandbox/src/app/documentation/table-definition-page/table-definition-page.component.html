TableDefinition

Provides definition of the table including following capabilities

  • Define columns
  • Define default sort column
  • Define row actions for rows
  • Define aggregation row
  • Define selectable rows mode
  • Define expandable children rows section for rows
  • Deine drag and drop mode
Example
Property Type Description
cols (Table.Column | Table.CustomColumn | TMapColumns)[] List of the columns definitions see bellow. Example
rowActions (optional) Table.RowAction[] | ((row: Table.Row) => Table.RowAction[]) Define list of actions associated with each row. Each row will have additional dropdown button component with provided actions. Instead of static array you could provide controller function which will take currently generated row and return array of the actions for each particular row, if this function will return empty array dropdown button won't be shown for the row.. This property intended to be used in conjuction with rowAction event. Example
details (optional) TableDetails This property provides interafce to populate children rows of the parent row.
sort (optional) string | Table.SortColumn Default sort column id (ascending order). With Table.SortColumn you could define not only column but also order.
aggregateRow Table.AggregateRow Display row with aggregated data from other rows. Data will be aggregated only for visible rows.
dragEnabled boolean (default false) Allow rows to be drag / dropped, see related drop event.
selectedRows (number | string)[] (default undefined) The list of the selected rows ids. If this value is undefined UI won't allow user select rows in the list, if this value is array (even empty), UI will display checkboxes which will allow user to select rows in the list. See related selectedRowsChanged event

Table.RowAction

Property Type Description
id any Action id, this property could be used on event handling.
title string Title of the action, it will be show on the dropdown item.

TableDetails

Property Type Description
cols (Table.Column | Table.CustomColumn | TMapColumns)[] Subset of root cols (by id), if cols not in the subset they will be ignored.
rows (parentRow: Table.Row) => any[] Function which given parent row should return array of children rows. Then children rows will be generated as sub rows of the parent.

Table.AggregateRow

Provides interface to define aggregation row on the table. The aggregated row will be shown first in the list and include aggregation data only from currently visible rows. For each column defined on the table you could provide aggregation function. This interface is the Map object `columnId : aggregationFunction for the column`. The aggregation function takes as a first param all values from all visible rows for this column and as second additional rows all visible rows. Example
