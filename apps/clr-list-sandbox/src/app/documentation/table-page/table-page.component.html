Table

Table component renders table UI representation by definition provided in table definition.

Table properties

Property Type Description
dataProvider Table.Data.DataProvider Provides interface to load data to the table.
defintion Table.Definition Define table UI representation: columns, sorting, etc in simple js object.

Table seconadry properties

These properties are for advanced cases and should be rarely used or changed from default values in most of the applications

Property Type Description
hidePaginator boolean (default false) Show / hide paginator for the table
disbaleSorting boolean (default false) Disable sorting for all columns even if they marked as sortable.
setFirstRowActiveOnFocus boolean (default true) Will set first row active when table component recieve focus.
rowSelectable boolean (default false) If this property set to true, then when some row clicked it will be highlighted as active. Usefull when open some item to display / edit in modal or sidepanel, this way it is clear for user which item from table is shown.
dragEnabled* recommended to use TableDescription.dragEnabled boolean (default false) Allow rows to be drag / dropped, see related drop event.
selectedRows* recommended to use TableDescription.selectedRows (number | string)[] (default undefined) The list of the selected rows ids. If this value is undefined UI won't allow user select rows in the list, if this value is array (even empty), UI will display checkboxes which will allow user to select rows in the list. See related selectedRowsChanged event
aggregateRow obsolete, use TableDescription.aggregateRow Table.AggregateRow Display row with aggregated data from other rows. Data will be aggregated only for visible rows.

* There is exacly same properties on Table and TableDescription object, the main difference here to understand table definition object used only once on component initialization, after this changes on this object is not reflected on the UI, most of the times its desired behaviour and you should use these properties on table definition object. But if for some reason you want to change these properties dynamically, use ones from table component.

Table events

Event Type Description
rowAction Table.RowActionEvent Will emit event when user click some item in row actions list. See related rowActions property on table definition object Example
cellClick Table.CellClickEvent Will emit event when user click on some row.
drop Table.DropEvent Will emit event when user dragged and dropped row. See related dragEnabled property.
selectedRowsCahnged Table.Row[] Will emit event when user select / unselect rows from selected list. selectedRows property.

Table child content

Content Description Example

hlcClrCustomCell Template with hlcClrRowDetail attribute which will be generated for row detail if root row have any. See related Table.CustomColumn type.
